# AGENTS.md

`mockups/<name>/` — self-contained HTML click-through mockups (inline CSS/JS, no build, no deps). Each dir has a `README.md` that IS the brief: how to render it, the design decisions (settled — don't relitigate), tried-and-rejected, and what's mocked vs. not.

- Working with a mockup? **Read that dir's `README.md` first.** It has the render command and the full context — don't reconstruct it by reading `index.html`.
- Reviewing one: render the key screens (command's in the README), then give a **tight** read — a few lines on what it is and whether it holds together. Don't spit a wall of context back; the README already carries the design rationale.
- Editing one: honor the README's settled decisions and house style. Don't relitigate them.
